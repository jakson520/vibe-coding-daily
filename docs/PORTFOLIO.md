# Vibe Coding Daily 作品集说明

## 1. 项目概述

Vibe Coding Daily 是一个面向 AI 编程学习者的个人信息聚合与每日推荐系统。它自动采集 AI 编程工具、Agent 开发、Codex、Cursor、Claude Code、GitHub Copilot 等相关内容，经过清洗、摘要、标签分类和推荐排序后，生成中文日报。

项目定位不是通用资讯站，而是一个垂直主题的信息雷达：帮助个人用户每天快速判断“今天 AI 编程领域有哪些内容值得看、为什么值得看、下一步可以做什么”。

## 2. 用户痛点

- 信息分散：工具更新、开源项目、教程和讨论分布在多个平台。
- 更新频率高：AI 编程工具迭代快，用户很难持续跟进。
- 筛选成本高：标题党、重复内容、低质量项目会消耗注意力。
- 英文内容多：学习者需要快速理解英文信息的核心价值。
- 缺少行动建议：很多资讯只告诉用户“发生了什么”，没有帮助用户判断“该不该跟进”。

## 3. 解决方案

系统将信息流处理拆成几个连续步骤：

1. 从多个可信来源采集最新内容。
2. 对内容进行去重、关键词过滤和质量过滤。
3. 自动打标签，区分工具更新、教程、项目案例、风险提醒和趋势观点。
4. 使用 AI 生成中文摘要和“为什么值得看”。
5. 结合相关度、新鲜度、来源可信度和热度计算推荐分。
6. 每日生成最多 10 条推荐，并支持页面查看、历史归档、收藏和邮件推送。

## 4. 核心流程

```txt
信息源配置
  -> 定时采集
  -> 内容清洗与去重
  -> 标签分类
  -> AI 摘要 / 本地降级摘要
  -> 推荐评分
  -> 日报生成
  -> 前端展示 / 邮件推送 / 日志记录
```

## 5. 功能模块

### 信息采集

内置 Hacker News、GitHub、Claude Code releases、Cursor changelog、VS Code updates、OpenAI News 等公开来源。每个 source 都记录最近抓取时间、状态、错误、抓取数量和入选数量。

### 内容处理

系统对采集结果进行标准化，保留标题、来源、链接、发布时间、原始摘要和热度，并基于内容哈希去重。

### 标签与分类

根据关键词和规则提取标签，例如 Cursor、Codex、AI Agent、GitHub Copilot、教程、风险和开源项目。

### 摘要与推荐

配置 OpenAI API 后使用 AI 生成中文摘要；未配置时使用本地降级摘要，保证系统可运行。推荐排序综合相关度、新鲜度、可信度和热度。

### 日报与展示

首页展示最新日报，历史页支持回看旧日报，用户可以收藏文章并按标签筛选。

### 后台与运行记录

`/admin/sources` 展示各信息源运行状态。每次日报任务会写入 `logs/daily-job.log`，方便验收和排查。

## 6. 产品亮点

- 垂直领域明确：聚焦 AI 编程和 Vibe Coding，而不是泛资讯。
- 输出结果面向行动：每条内容都有中文摘要和“为什么值得看”。
- 可降级运行：没有 OpenAI API Key 时仍能生成日报。
- 可观测性基础完整：source 状态和 daily-job 日志让失败原因可追踪。
- MVP 范围清晰：先完成个人可用的信息流闭环，再规划云端化和个性化。

## 7. 技术实现概览

- 前端：Next.js App Router、TypeScript、Tailwind CSS
- 数据层：Prisma schema + SQLite 本地数据库
- 采集层：按 source 类型拆分 collector，统一输出标准文章结构
- 摘要层：OpenAI Responses API + 本地 fallback
- 排序层：相关度、新鲜度、来源可信度、热度加权
- 推送层：Nodemailer SMTP 邮件框架
- 调度层：`node-cron` 定时脚本和手动触发 API
- 后台：source 状态只读页面

## 9. 项目难点与解决方式

### 信息源稳定性

不同平台 API、网页结构和权限限制不同。项目通过 source 状态字段记录 `success / failed / skipped`，即使单个源失败，也不会阻塞整份日报生成。

### 摘要可靠性

AI 摘要可能出现不准确或格式不稳定。项目限制输出字段，并保留本地降级摘要，确保没有 API Key 或 API 调用失败时系统仍能工作。

### 推荐标准

资讯“值得看”不是单一指标。项目将相关度、新鲜度、来源可信度和热度拆开计算，便于后续根据用户反馈调整权重。

### MVP 边界

当前版本没有做复杂用户系统和多租户，而是优先完成个人可用的端到端闭环：采集、处理、推荐、展示、日志和后台状态。

## 10. 后续迭代方向

- 使用 Supabase / Postgres 替代本地 SQLite
- 增加用户兴趣配置和关键词管理
- 支持更多内容源和 RSS 自定义
- 增加趋势分析和主题聚类
- 增加周报、月报和年度回顾
- 部署上线并接入稳定的云端定时任务
- 增加后台鉴权和信息源开关管理
